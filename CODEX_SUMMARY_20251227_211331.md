# Repository Summary (2025-12-27 21:13:31)

## What this repo is
This is a hybrid compression + filesystem research project centered on **PSFS** (PermStream File System) and **PermStream Pro** compression. It combines a custom compression pipeline (predictor + permutation rank + arithmetic coding) with a **mountable, immutable container format** and an **append-only journal overlay** for writes.

## Core components

### PermStream Pro compression (`permstream_pro.py`)
Implements the compression algorithm, stream modes, and CLI for compress/decompress. It also contains PSFS pack/unpack/verify functions used by the filesystem tools.

### PSFS container format
A random-access immutable container (like a “frozen” filesystem).
- Pack a directory → `.psfs` container
- Unpack a container → directory
- Verify container integrity

### PSFJ journal overlay (sidecar)
An append-only journal that overlays changes on top of the base container without modifying it.
- Supports files, directories, symlinks, and deletes
- Conflict resolution by timestamp + append order
- Enables write support without rewriting the base container

### PSFB blob file (sidecar for large writes)
Large file writes are stored as **extents** in a blob file; the journal stores extent metadata + a commit record. This avoids loading multi-GB data into memory and supports large CSV/media workloads.

## Filesystem support

### FUSE mount (`psfs_fuse.py`)
- Read-only mount of `.psfs` with optional journal overlay
- Write support via journal (create/write/rename/delete/etc.)
- Caching, readahead, CRC checks
- Journal stats on unmount
- Large-file support via extents + blob file
- Configurable durability and compression heuristics

## Tooling (unified CLI)

### `psfs.py`
- `pack`, `unpack`, `verify`
- `mount` with journal and profiles
- `journal` helpers (add/mkdir/symlink/delete/stats/max-mtime)
- `compact` (merge base + journal, incremental, watermark)
- `fsck` (validate, conflicts, repair)

## Operational tools

### Compaction (`psfs_compact.py`)
Merges base container + journal → new `.psfs`. Supports incremental compaction and watermarking.

### FSCK (`psfs_fsck.py`)
Validates containers and journals, reports conflicts, and can repair journals (copy-out or in-place with backups).

### Benchmarking (`bench_journal.py`)
Measures journal write performance vs fsync cadence and compression thresholds; supports CSV/JSON output.

## Design posture
- Immutable base container + append-only journal = crash-safe write model.
- Large files are stored as blob extents to avoid RAM blow-up.
- Tunable durability/compression tradeoffs via **profiles** (`business`, `media`) and explicit flags.

## In short
This repo is building a **high-performance, compression-aware, append-only filesystem overlay** that lets you pack directories into a compressed container, mount them read-only, and apply writable changes through a durable journal—with a path to handle multi-GB CSV/media workloads safely and efficiently.
