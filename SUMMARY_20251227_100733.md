# PSFS Work Summary (2025-12-27 10:07:33)

## Overview
This update series built out PSFS as a functional, journaled, mountable filesystem layer with a compact-on-demand model. Work focused on correctness (dir/symlink handling, journal overlay, conflict rules), operational tooling (compact, fsck, stats, repair), and durability/perf controls (fsync cadence, compression heuristics, cache stats), with a unified CLI for all tasks.

## Major Features Implemented

### PSFS Container + Format Enhancements
- Added directory and symlink support in pack/unpack and FUSE overlay; symlink payloads store link target bytes.
- Updated `FORMAT.md` with PSFJ journal spec and conflict-resolution policy.
- Fixed codec mismatch: entropy quantization now applied before arithmetic encode; corrected no-rank decode bug.

### Journal Overlay (PSFJ)
- Implemented PSFJ append-only journal format and writer:
  - File, dir, symlink, and delete records.
  - Optional payload compression in journal records.
  - Delete records carry current `mtime_ns`.
- FUSE overlay supports journal entries with version ordering (latest `mtime_ns`, tie by append order).
- Recursive deletes are tracked and mask older entries.

### Compaction + Incremental Compaction
- `psfs_compact.py` merges base PSFS + journal into a new PSFS.
- Added incremental compaction via `--since-ns` to apply only records >= a cutoff.
- Added `--truncate-journal` to reset journal to header after compaction.
- Added `--watermark` to read/write last-applied `mtime_ns` for automatic incremental compaction.

### FSCK + Repair + Conflict Diagnostics
- `psfs_fsck.py` validates base PSFS and journal entries; deep mode can decompress payloads.
- Added conflict diagnostics (`--conflicts`) to report paths with multiple records and winner by policy.
- Added journal repair (`--repair-journal`) to copy out only valid records and drop bad ones.

### FUSE Mount + Write Support
- FUSE read path includes LRU cache, readahead, CRC validation, and stats on unmount.
- Write support via PSFJ overlay: create/write/truncate/unlink/mkdir/rmdir/rename/symlink/chmod/chown/utimens.
- Added `mknod` support for O_CREAT flows from shell tools.
- Reads prefer open write-handle buffers when present.

### Durability + Compression Heuristics
- Journal fsync options:
  - `--journal-sync` (fsync every write)
  - `--journal-sync-every N` (fsync every N records)
- Compression heuristics for journal payloads:
  - `--journal-compress-min-size` (default 4096)
  - `--journal-compress-min-gain` (default 128)
- Added journal write stats on unmount (`records`, `raw bytes`, `stored bytes`, `compressed records`, `fsyncs`).

### Unified CLI (`psfs.py`)
- One CLI covers pack/unpack/verify/mount/journal/compact/fsck.
- Added new subcommands/flags:
  - `journal max-mtime`
  - `journal stats [--json]`
  - `compact --since-ns --truncate-journal --watermark`
  - `fsck --conflicts --conflicts-limit --repair-journal`
  - mount flags for durability/compression tuning
- Lazy-imported FUSE/`psfs_fuse` to avoid crashes when fuse is not available.

## Behavioral/Policy Notes
- Conflict resolution: newest `(mtime_ns, append order)` wins.
- Recursive deletes mask older entries; masking uses the same version ordering.
- Journal compression is optional and only retained if it meets size/gain thresholds.

## Known Constraints / Open Decisions

- Durability defaults (sync cadence vs buffered) are configurable but not tuned yet.
- Performance profiling + cache tuning still pending.
